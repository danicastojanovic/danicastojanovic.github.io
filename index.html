projects

Catwalk

I architected a scalable backend for an existing e-commerce site. I worked on the questions and answers microservice that had millions of records, towards integrating with an existing front-end. The process of improving performance and increasing throughput was driven with quantitative data derived from continuous testing using Loader.io that helped me understand the ROI and bottlenecks of the system. The API-first approach also enabled me to organize the backend to separate concerns and group queries using Express routing.

Djikstra

I was tasked with redesigning the related items and outfit carousel widget of a retail e-commerce application.
One of the challenges was building a dynamic scroll feature where a maximum of four product cards would be displayed, and arrow buttons would scroll one image at a time. Using vanilla JS, calculations of the width and position of the element, and by using the 'scrollby' method, I was able to implement this design feature.
Another feature was the product comparison modal that involved creating a neat table for easy overview. Finally, the star rating was a fun component built by overlaying unicode with dynamic element width (for reusability in other widgets).

Wherewolf

If you have ever player the party game "Werewolf", this is a web app that simulates this gameplay, with the advantage of the back-end keeping track of the gameplay.
This relatively complex game logic was enabled by using object-oriented design, with objects encapsulating the properties of different players, the game, and game progress. In order to communicate these states with the players, as well as allow each player's input to alter the game state, the use of sockets came in pretty handy (event emitters and listeners).
